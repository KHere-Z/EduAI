> **项目名称**: EduAI · 全学科 AI 智能教育平台  
> **版本**: v1.0  
> **日期**: 2026-06-30  
> **预计用户**: 5,000+ 并发

---

## 一、项目概述

### 1.1 项目背景

传统教育平台学科割裂、缺乏智能化。EduAI 是一个覆盖 **9 大学科** 的 AI 智能教育系统，为教师和学生提供从错题管理到 AI 综合分析的全链路学习支持。

### 1.2 核心目标

| 维度 | 目标 |
|------|------|
| 学科覆盖 | 语文 / 数学 / 英语 / 物理 / 化学 / 生物 / 历史 / 政治 / 地理 |
| AI 赋能 | 错题举一反三、AI 课堂反馈、AI 综合分析、智能题库 |
| 规模支撑 | 5,000+ 并发用户，毫秒级响应 |
| 多端适配 | 教练端 / 学生端 / 机构端 / 平台管理端 |

## 四、功能详细规格

### 4.1 错题整理

| 属性 | 规格 |
|------|------|
| 录入方式 | 手动录入 / 拍照 OCR / 试卷扫描导入 |
| 分类维度 | 学科 → 知识点 → 错误类型 → 难度 |
| 标签系统 | 高频错误 / 概念混淆 / 计算失误 / 审题不清 |
| 搜索 | 按关键字、日期范围、知识点筛选 |
| 导出 | PDF 错题本 / 打印试卷 |

### 4.2 错题分析 + 举一反三

```
流程:
  错题录入 → AI 分析错误根因 → 生成知识点诊断
                ↓
  举一反三 ← 同类题推荐 ← 变式题生成
```

| 属性 | 规格 |
|------|------|
| AI 模型 | DeepSeek (已通过 Headroom 接入) |
| 举一反三 | 同知识点变式 3 题 + 综合运用 1 题 |
| 分析维度 | 知识点缺失 / 思维偏差 / 粗心 / 概念混淆 |
| 输出 | 错误根因 + 推荐复习内容 + 变式练习题 |

### 4.3 知识点整理

- 按 "学段 → 年级 → 学期 → 单元 → 知识点" 五级树
- 每个知识点包含：定义、例题、关联考点、掌握度
- 支持自定义知识点图谱

### 4.4 考点整理

- 标注近 5 年考试出现频次（高频/中频/低频）
- 关联历年真题
- 每个考点标注难度星级和分值占比

### 4.5 解题模型

| 数学 | 物理 | 化学 | 其他 |
|------|------|------|------|
| 函数模型 | 力学模型 | 离子反应 | 阅读理解模板 |
| 几何模型 | 电学模型 | 有机合成 | 作文框架 |
| 概率模型 | 光学模型 | 计算模型 | 材料分析 |

### 4.6 题库

| 属性 | 规格 |
|------|------|
| 题目来源 | 历年真题 + AI 生成 + 教师上传 |
| 题目属性 | 学科 / 知识点 / 题型 / 难度 / 来源 |
| AI 生成 | 基于知识点自动生成变式题 |
| 检索 | 多条件组合筛选 + 模糊搜索 |

### 4.7 AI 课堂反馈

| 反馈维度 | 说明 |
|----------|------|
| 知识点覆盖率 | 本节课覆盖了多少知识点 |
| 学生参与度 | 答题正确率 / 活跃度 |
| 薄弱点预警 | AI 识别班级共性问题 |
| 教学建议 | AI 生成补强方案 |

### 4.8 AI 综合分析

- **能力雷达图**: 各知识点掌握度可视化
- **学情趋势**: 周/月/学期进步曲线
- **薄弱项诊断**: AI 定位具体知识缺口
- **冲刺建议**: 考试前重点复习清单

### 4.9 成绩统计分析

| 维度 | 指标 |
|------|------|
| 个人 | 得分 / 排名 / 进步幅度 / 知识点得分率 |
| 班级 | 平均分 / 最高分 / 最低分 / 标准差 / 分段统计 |
| 年级 | 横向对比 / 正态分布图 |

#### Spring Boot 分层架构

```
┌─────────────────────────────────────────┐
│  Controller  层  (@RestController)      │  ← 参数校验、VO 转换
├─────────────────────────────────────────┤
│  Service     层  (@Service)             │  ← 业务逻辑、事务管理
├─────────────────────────────────────────┤
│  Repository  层  (@Repository / Mapper) │  ← 数据访问
├─────────────────────────────────────────┤
│  Domain      层  (@Entity / Model)      │  ← 实体定义
└─────────────────────────────────────────┘
     ↕
  AOP 横切: 日志 / 权限 / 缓存 / 限流
```

#### 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| ORM | JPA + MyBatis-Plus 混用 | 简单 CRUD 用 JPA，复杂报表用 MyBatis |
| 认证 | Sa-Token + JWT | 比 Security 上手快 3 倍，功能不输 |
| AI 异步 | RocketMQ | 举一反三/综合分析耗时长，异步返回 |
| 文件存储 | MinIO（开发）→ 阿里云 OSS（生产） | S3 兼容，切换 0 成本 |
| 对象映射 | MapStruct | 编译期生成，比 BeanUtils 快 10-20 倍 |
| API 文档 | Knife4j | 国产 Swagger 增强、中文友好 |
